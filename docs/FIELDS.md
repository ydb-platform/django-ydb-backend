Fields
===

The YDB backend supports most built-in Django fields. This page gives the
support level for each field and the exact YDB type mapping.

## Support at a glance

| Field | Status | Notes |
|-------|:------:|-------|
| `AutoField`, `SmallAutoField`, `BigAutoField` | ✅ | `Serial` family; the value is generated by YDB and read back via `INSERT ... RETURNING`. |
| `CharField`, `TextField`, `SlugField`, `EmailField` | ✅ | Stored as `Utf8`. |
| `IPAddressField`, `GenericIPAddressField` | ✅ | Stored as `Utf8`. |
| `BinaryField`, `FileField`, `FilePathField` | ✅ | `String` / `Utf8`. |
| `IntegerField` and the `Small`/`Big`/`Positive*` variants | ✅ | Signed or unsigned integer types chosen by range. |
| `BooleanField` | ✅ | `Bool`. |
| `FloatField` | ✅ | `Float`; cannot be a primary key. |
| `DecimalField` | ✅ | Arbitrary `max_digits` / `decimal_places`; values are rounded to the field's scale (YDB keeps roughly 26 significant digits). |
| `UUIDField` | ✅ | Native `UUID`. |
| `DurationField` | ✅ | `Interval` (±136 years); cannot be a primary key. |
| `DateField` | ✅ | Signed wide date type, so dates before 1970 round-trip. |
| `DateTimeField` (`USE_TZ=True`) | ✅ | Signed wide timestamp with microsecond precision; instants before 1970 round-trip. |
| `DateTimeField` (`USE_TZ=False`) | 🟡 | Naive datetimes shift by the server timezone on round-trip. The production default `USE_TZ=True` is unaffected. |
| `TimeField` | 🟡 | YDB has no native time type; stored as microseconds since midnight and introspected back as `BigIntegerField`. `__hour` / `__minute` / `__second` lookups work. |
| `JSONField` | 🟡 | Native `Json`. Equality filtering (`filter(data=value)`) is not supported by YDB and the column cannot be introspected. `null=True` works, and `__isnull` filtering works. |

## Relationship fields

[ForeignKey](https://docs.djangoproject.com/en/dev/ref/models/fields/#foreignkey),
[ManyToManyField](https://docs.djangoproject.com/en/dev/ref/models/fields/#manytomanyfield),
and [OneToOneField](https://docs.djangoproject.com/en/dev/ref/models/fields/#onetoonefield)
work through the Django ORM, but **YDB does not enforce foreign-key or uniqueness
constraints at the database level**.

| Relation feature | Status | Notes |
|------------------|:------:|-------|
| `ForeignKey`, `OneToOneField` | ✅ | Stored as a scalar `<name>_id` column typed from the target's primary key. |
| `ManyToManyField` (auto-through and `through=`) | ✅ | Through tables are ordinary YDB tables; add / list / remove works. |
| `select_related` | ✅ | |
| `prefetch_related` | 🟡 | Works; a few edge cases fail. |
| Database-level foreign keys (`REFERENCES`, `ON DELETE`) | ❌ | Not emitted and not introspected. |
| Database-level cascade delete | ❌ | Django's ORM `on_delete` still runs for ORM deletes, but raw SQL or external writers can orphan rows. |
| One-to-one / many-to-many-pair uniqueness | ❌ | Not enforced by YDB; the ORM declaration is accepted but duplicates are not rejected. |

The backend stores `ForeignKey` values as regular scalar columns. For example:

```python
class Product(models.Model):
    sku = models.CharField(max_length=20, primary_key=True)


class ProductReview(models.Model):
    product = models.ForeignKey(Product, on_delete=models.CASCADE)
    rating = models.IntegerField()
```

`ProductReview.product` is stored as a `product_id` column with the same YDB type
as `Product.sku`. The schema does not include `FOREIGN KEY`, `REFERENCES`,
database-level `ON DELETE` rules, or unique constraints for relation fields.

This means inserts and updates write the referenced primary key value only:

```python
ProductReview.objects.create(product_id="SKU-1", rating=5)
```

YDB accepts this row even if no `Product(sku="SKU-1")` exists. Django ORM
cascade behavior applies only when deleting objects through Django. Raw SQL,
external writers, or direct `*_id` assignments may create orphan rows.

`OneToOneField` is stored like a `ForeignKey`; its one-to-one uniqueness is not
guaranteed by YDB. `ManyToManyField` uses a Django through table with scalar
`*_id` columns, but uniqueness of pairs and database-level cascade behavior are
not enforced. Enforce these in application code (`validate_unique()`, `clean()`,
validators) where you need them.

## Type mapping

The exact YDB type each Django field maps to:

| Class                                      | DB Type  | Pythonic Type     | Comments                                                                                                                                             |
|--------------------------------------------|----------|-------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|
| django.db.models.SmallAutoField            | Int16    | int               | YDB type `SmallSerial` generates the value automatically. Range -32768 to 32767                                                                      |
| django.db.models.AutoField                 | Int32    | int               | YDB type `Serial` generates the value automatically. Range -2147483648 to 2147483647                                                                 |
| django.db.models.BigAutoField              | Int64    | int               | YDB type `BigSerial` generates the value automatically. Range -9223372036854775808 to 9223372036854775807                                            |
| django.db.models.CharField                 | UTF-8    | str               |                                                                                                                                                      |
| django.db.models.TextField                 | UTF-8    | str               |                                                                                                                                                      |
| django.db.models.BinaryField               | String   | bytes             |                                                                                                                                                      |
| django.db.models.SlugField                 | UTF-8    | str               |                                                                                                                                                      |
| django.db.models.FileField                 | String   | bytes             |                                                                                                                                                      |
| django.db.models.FilePathField             | UTF-8    | str               |                                                                                                                                                      |
| django.db.models.DateField                 | Date32      | datetime.date     | Signed wide date type, so dates before 1970 round-trip (range ~144169 BCE to 148107 CE)                                                              |
| django.db.models.DateTimeField             | Timestamp64 | datetime.datetime | Signed wide type with microsecond precision; supports instants before 1970 (range ~144169 BCE to 148107 CE)                                          |
| django.db.models.TimeField                 | Int64       | datetime.time     | YDB has no native time type; stored as microseconds since midnight (introspects back as BigIntegerField)                                             |
| django.db.models.DurationField             | Interval | int               | Range from -136 years to +136 years. Internally a 64-bit signed integer. Cannot be used in the primary key                                           |
| django.db.models.SmallIntegerField         | Int16    | int               | Range -32768 to 32767                                                                                                                                |
| django.db.models.IntegerField              | Int32    | int               | Range -2147483648 to 2147483647                                                                                                                      |
| django.db.models.BigIntegerField           | Int64    | int               | Range -9223372036854775808 to 9223372036854775807                                                                                                    |
| django.db.models.PositiveSmallIntegerField | UInt16   | int               | Range 0 to 65535                                                                                                                                     |
| django.db.models.PositiveIntegerField      | UInt32   | int               | Range 0 to 4294967295                                                                                                                                |
| django.db.models.PositiveBigIntegerField   | UInt64   | int               | Range 0 to 18446744073709551615                                                                                                                      |
| django.db.models.FloatField                | Float    | float             | A real number with variable precision, 4 bytes in size. Cannot be used in the primary key                                                            |
| django.db.models.DecimalField              | Decimal  | Decimal           | Column and bound parameters use the field's `max_digits` / `decimal_places`. Values are rounded to the field's scale; YDB keeps ~26 significant digits. |
| django.db.models.UUIDField                 | UUID     | uuid.UUID         |                                                                                                                                                      |
| django.db.models.IPAddressField            | UTF-8    | str               |                                                                                                                                                      |
| django.db.models.GenericIPAddressField     | UTF-8    | str               |                                                                                                                                                      |
| django.db.models.BooleanField              | Bool     | bool              |                                                                                                                                                      |
| django.db.models.EmailField                | UTF-8    | str               |                                                                                                                                                      |
| django.db.models.JSONField                 | Json     | dict / list / str / int / float / bool | Native YDB `Json` type. Equality filtering (`filter(data=value)`) is not supported by YDB. `null=True` works (stored as SQL NULL; `__isnull` filtering works). |
